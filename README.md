# Quantifying Model Collapse in Small Code Models

This repository gives you a compressed, runnable version of the paper idea:

- build a clean human-written code corpus
- define tail tokens by frequency
- create a synthetic head-domain corpus
- continue pretraining a small causal LM under different clean/synthetic mixtures
- measure tail-token log-probability and tail-only perplexity on held-out human code

The defaults are tuned for a fast project submission:

- base model: `distilgpt2` by default
- optional upgrade: `gpt2`
- data source: sampled code from Hugging Face datasets or local files
- synthetic corpus: template-generated by default, with optional model/API generation

## Quick Start

### 1. Create environment

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

If you want optional OpenAI-based synthetic generation:

```bash
pip install -e .[openai]
```

### 2. Prepare clean corpus

Option A: Hugging Face dataset.

Use **The Stack**, which is the dataset proposed in the paper:

- dataset: `bigcode/the-stack-dedup`
- text column: `content`
- language column: `lang`
- language shards: `data/python`, `data/javascript`, `data/typescript`, `data/java`, `data/c`, `data/cpp`, `data/go`, `data/rust`, `data/solidity`, `data/cuda`, `data/assembly`, `data/shell`

Important: The Stack is gated on Hugging Face. You must accept the dataset terms on the dataset page and be logged in locally before running the pipeline.

This repo is written so you can point it at a dataset that contains code examples. A good first try is a code dataset you can stream from `datasets`. Example:

```bash
python scripts/inspect_hf_dataset.py \
  --dataset-name bigcode/the-stack-dedup \
  --dataset-data-dir data/python
```

Use that output to confirm the right split, config, text column, and language column, then run:

```bash
python scripts/prepare_clean_corpus.py \
  --dataset-name bigcode/the-stack-dedup \
  --dataset-data-dir data/python \
  --dataset-data-dir data/javascript \
  --dataset-data-dir data/typescript \
  --dataset-data-dir data/java \
  --dataset-data-dir data/c \
  --dataset-data-dir data/cpp \
  --dataset-data-dir data/go \
  --dataset-data-dir data/rust \
  --dataset-data-dir data/solidity \
  --dataset-data-dir data/cuda \
  --dataset-data-dir data/assembly \
  --dataset-data-dir data/shell \
  --text-column content \
  --language-column lang \
  --languages Python JavaScript TypeScript Java C C++ Go Rust Solidity CUDA Assembly Shell \
  --language-quota Python=0.21 \
  --language-quota JavaScript=0.15 \
  --language-quota TypeScript=0.13 \
  --language-quota Java=0.12 \
  --language-quota C=0.08 \
  --language-quota C++=0.08 \
  --language-quota Go=0.06 \
  --language-quota Rust=0.05 \
  --language-quota Solidity=0.05 \
  --language-quota CUDA=0.04 \
  --language-quota Assembly=0.02 \
  --language-quota Shell=0.01 \
  --tail-eval-language Java \
  --tail-eval-language C \
  --tail-eval-language C++ \
  --tail-eval-language Go \
  --tail-eval-language Rust \
  --tail-eval-language Solidity \
  --tail-eval-language CUDA \
  --tail-eval-language Assembly \
  --max-train-samples 12000 \
  --max-eval-samples 800 \
  --output-dir artifacts/data
```

If that dataset name or columns differ in your environment, run with `--help` and update the arguments. The script is intentionally generic.

Option B: local code files.

```bash
python scripts/prepare_clean_corpus.py \
  --input-glob "/path/to/code/**/*.py" \
  --input-glob "/path/to/code/**/*.js" \
  --input-glob "/path/to/code/**/*.sh" \
  --max-train-samples 12000 \
  --max-eval-samples 800 \
  --output-dir artifacts/data
```

This writes:

- `artifacts/data/clean_train.jsonl`
- `artifacts/data/tail_test.jsonl`
- `artifacts/data/tail_tokens.json`
- `artifacts/data/clean_stats.json`

If one language dominates the stream too heavily, use `--language-quota` to collect a weighted mix instead of pure arrival order. The weights do not need to sum to 1.0; they are normalized internally.

To avoid treating odd comments and arbitrary rare strings as “tail,” the default pipeline can instead hold out a rare-language slice with `--tail-eval-language` and then score all token positions in that slice.

### 3. Generate synthetic head-domain corpus

Default path: OpenAI backend.

```bash
python scripts/generate_synthetic_head.py \
  --backend openai \
  --generator-model gpt-4o-mini \
  --target-samples 12000 \
  --output-path artifacts/data/synth_head.jsonl
```

This generates 12,000 synthetic code samples by default, focused on common head-domain patterns such as:

- React
- Next.js
- FastAPI
- Flask
- common Python scripts and utilities
- common JavaScript/Node.js app patterns

Optional: use a Hugging Face model to generate synthetic code.

```bash
python scripts/generate_synthetic_head.py \
  --backend hf \
  --generator-model distilgpt2 \
  --target-samples 4000 \
  --output-path artifacts/data/synth_head.jsonl
```

Optional: use OpenAI API if you have a key.

```bash
export OPENAI_API_KEY=...
python scripts/generate_synthetic_head.py \
  --backend openai \
  --generator-model gpt-4o-mini \
  --target-samples 4000 \
  --output-path artifacts/data/synth_head.jsonl
```

### 4. Run the experiment

```bash
python scripts/run_experiment.py --config configs/experiment.yaml
```

This will train three runs:

- `100_clean_0_synth`
- `75_clean_25_synth`
- `50_clean_50_synth`

Outputs land in `artifacts/runs/<run_name>/`.

The pipeline also writes paper-ready summaries into:

- `artifacts/paper/methodology_summary.md`
- `artifacts/paper/results_summary.md`
- `artifacts/paper/artifact_manifest.json`

### 5. Plot results

```bash
python scripts/plot_results.py \
  --results-glob "artifacts/runs/*/tail_metrics.json" \
  --output-dir artifacts/plots
```

## What To Expect

The thesis you are testing is:

> as the proportion of synthetic head-domain code increases, the model assigns lower probability to held-out tail tokens from human-written code.

The two main metrics are:

- mean tail-token log-probability
- tail-only perplexity

If the thesis is supported, then as contamination rises:

- log-probability becomes more negative
- tail-only perplexity increases

## Practical Notes

For a two-week class project, use these settings first:

- `distilgpt2`
- `max_train_steps: 500 to 1500`
- `per_device_train_batch_size: 2 to 4`
- `gradient_accumulation_steps: 8 to 16`
- `block_size: 128`
- `max_train_samples: 8k to 20k`

That is enough to show a trend without pretending to run a large-scale continued pretraining study.

## If Hugging Face Download Fails

Use local files instead. The clean-data script accepts multiple `--input-glob` arguments and will build the corpus from your machine directly. If you want, put permissively licensed code into a directory like:

```text
data/raw_code/
  python/
  javascript/
  shell/
```

and run:

```bash
python scripts/prepare_clean_corpus.py \
  --input-glob "data/raw_code/**/*.py" \
  --input-glob "data/raw_code/**/*.js" \
  --input-glob "data/raw_code/**/*.sh" \
  --output-dir artifacts/data
```

## Suggested Report Framing

Be explicit that this is a scaled-down empirical proxy of the proposal:

- small model instead of LLaMA 3.2 3B
- sampled code corpus instead of the full Stack
- synthetic head-domain approximated with templated or generated common code tasks

That is still valid for demonstrating the directional effect.
